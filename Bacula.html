Backup Server with Bacula

Bacula is an enterprise-level, open-source network backup solution that allows you to backup, restore, and verify data across your network. This tutorial covers setting up a Bacula backup server on Ubuntu to protect your critical data.

Prerequisites

  • Ubuntu Server 20.04 LTS or newer
  • Minimum 4GB RAM (8GB recommended for large deployments)
  • Significant disk space for backup storage (size depends on your needs)
  • Root or sudo privileges
  • Static IP address configured
  • MySQL or PostgreSQL database server

1 System Preparation

Update your system and install required dependencies:

sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential wget curl

Set the hostname for your backup server:

sudo hostnamectl set-hostname bacula-server
exec bash

2 Install MySQL Database

Install MySQL server for Bacula's catalog:

sudo apt install -y mysql-server mysql-client

Secure your MySQL installation:

sudo mysql_secure_installation
Note: Follow the prompts to set a root password and secure your database installation.

3 Install Bacula Components

Add the Bacula repository and install all components:

sudo apt install -y bacula-server bacula-client bacula-common \
bacula-mysql bacula-console

During installation, you'll be prompted to configure database for bacula-director-mysql. Select "Yes" and provide the MySQL root password.

Set a password for the Bacula database user when prompted.

4 Configure MySQL Database

Log into MySQL and verify the Bacula database was created:

sudo mysql -u root -p -e "SHOW DATABASES;"

You should see the bacula database in the list.

Verify the Bacula database tables:

sudo mysql -u root -p -e "USE bacula; SHOW TABLES;"

5 Configure Bacula Director

The Director daemon controls all backup, restore, verify, and archive operations.

Edit the main Bacula Director configuration file:

sudo nano /etc/bacula/bacula-dir.conf

Update the Director resource to match your environment:

Director {
  Name = bacula-dir
  DIRport = 9101
  QueryFile = "/etc/bacula/query.sql"
  WorkingDirectory = "/var/lib/bacula"
  PidDirectory = "/run/bacula"
  Maximum Concurrent Jobs = 20
  Password = "your_director_password"
  Messages = Daemon
}

Update the Console resource for local administration:

Console {
  Name = bacula-mon
  Password = "your_console_password"
  CommandACL = status, .status
}

6 Configure Storage Daemon

The Storage Daemon performs the storage and recovery of files.

Edit the Storage Daemon configuration file:

sudo nano /etc/bacula/bacula-sd.conf

Verify the Storage resource configuration:

Storage {
  Name = bacula-sd
  SDPort = 9103
  WorkingDirectory = "/var/lib/bacula"
  PidDirectory = "/run/bacula"
  Maximum Concurrent Jobs = 20
  SDAddress = 127.0.0.1
}

# Define storage device (File storage)
Device {
  Name = FileStorage
  Media Type = File
  Archive Device = /bacula/backup
  LabelMedia = yes
  Random Access = yes
  AutomaticMount = yes
  RemovableMedia = no
  AlwaysOpen = no
}

Create the backup storage directory:

sudo mkdir -p /bacula/backup
sudo chown -R bacula:bacula /bacula
sudo chmod -R 700 /bacula

7 Configure File Daemon

The File Daemon (bacula-fd) runs on clients and provides file backup and restore services.

Edit the File Daemon configuration file:

sudo nano /etc/bacula/bacula-fd.conf

Verify the File Daemon configuration:

Director {
  Name = bacula-dir
  Password = "your_client_password"
}

FileDaemon {
  Name = bacula-fd
  FDport = 9102
  WorkingDirectory = /var/lib/bacula
  PidDirectory = /run/bacula
  Maximum Concurrent Jobs = 20
  FDAddress = 0.0.0.0
}

Messages {
  Name = Standard
  director = bacula-dir = all, !skipped, !restored
}

8 Configure Console

The Console allows you to interact with the Director.

Edit the Console configuration file:

sudo nano /etc/bacula/bconsole.conf

Verify the Director connection settings:

Director {
  Name = bacula-dir
  DIRport = 9101
  address = localhost
  Password = "your_console_password"
}

9 Start and Enable Bacula Services

Start all Bacula services and enable them to start on boot:

sudo systemctl start bacula-director bacula-sd bacula-fd
sudo systemctl enable bacula-director bacula-sd bacula-fd

Check the status of all services:

sudo systemctl status bacula-director
sudo systemctl status bacula-sd
sudo systemctl status bacula-fd

10 Test Bacula Configuration

Test the Bacula configuration files for errors:

sudo bacula-dir -t /etc/bacula/bacula-dir.conf
sudo bacula-sd -t /etc/bacula/bacula-sd.conf
sudo bacula-fd -t /etc/bacula/bacula-fd.conf

All commands should return "Config OK" if there are no errors.

11 Configure Firewall

Allow necessary ports through the firewall:

sudo ufw allow 9101/tcp  # Director
sudo ufw allow 9102/tcp  # File Daemon
sudo ufw allow 9103/tcp  # Storage Daemon
sudo ufw enable

Verify the firewall status:

sudo ufw status

12 Using Bacula Console

Access the Bacula console to manage backups:

sudo bconsole

Once in the console, you can run various commands:

status director    # Check director status
status storage     # Check storage status
show jobs          # List all jobs
run                # Run a backup job
messages           # Show messages
quit               # Exit the console

13 Create a Backup Job

In the Bacula Director configuration, add a backup job for your system:

sudo nano /etc/bacula/bacula-dir.conf

Add a new Job resource:

Job {
  Name = "BackupServer"
  Type = Backup
  Level = Incremental
  Client = bacula-fd
  FileSet = "Full Set"
  Schedule = "WeeklyCycle"
  Storage = FileStorage
  Messages = Standard
  Pool = Default
  Priority = 10
  Write Bootstrap = "/var/lib/bacula/%c.bsr"
}

Define a FileSet for what to backup:

FileSet {
  Name = "Full Set"
  Include {
    Options {
      signature = MD5
      compression = GZIP
    }
    File = /etc
    File = /home
    File = /var/www
  }
  Exclude {
    File = /var/lib/bacula
    File = /tmp
    File = /proc
    File = /sys
  }
}

Restart the Director to apply changes:

sudo systemctl restart bacula-director

14 Run Your First Backup

Start the Bacula console and run a backup:

sudo bconsole

In the console, run the backup job:

run
yes

Check the job status:

status director
messages

Wait for the job to complete and verify it was successful.

15 Configure Remote Clients

To backup remote Linux clients, install the Bacula client package:

# On the remote client
sudo apt install -y bacula-client bacula-fd

Configure the File Daemon on the client:

sudo nano /etc/bacula/bacula-fd.conf

Update the Director section with your Bacula server's IP:

Director {
  Name = bacula-dir
  Password = "your_client_password"
  Address = your_bacula_server_ip
}

On the Bacula server, add the client to the Director configuration:

sudo nano /etc/bacula/bacula-dir.conf

Add a new Client resource:

Client {
  Name = remote-client-fd
  Address = remote_client_ip
  FDPort = 9102
  Catalog = MyCatalog
  Password = "your_client_password"
  File Retention = 30 days
  Job Retention = 6 months
  AutoPrune = yes
}

16 Advanced Configuration

Set up backup pools for different retention policies:

# Define a pool for daily backups with short retention
Pool {
  Name = Daily
  Pool Type = Backup
  Recycle = yes
  AutoPrune = yes
  Volume Retention = 7 days
  Maximum Volume Jobs = 5
  Maximum Volumes = 10
}

# Define a pool for weekly backups with medium retention
Pool {
  Name = Weekly
  Pool Type = Backup
  Recycle = yes
  AutoPrune = yes
  Volume Retention = 30 days
  Maximum Volume Jobs = 4
  Maximum Volumes = 4
}

# Define a pool for monthly backups with long retention
Pool {
  Name = Monthly
  Pool Type = Backup
  Recycle = yes
  AutoPrune = yes
  Volume Retention = 365 days
  Maximum Volume Jobs = 12
  Maximum Volumes = 12
}

Configure different schedules for backup levels:

Schedule {
  Name = "WeeklyCycle"
  Run = Full 1st sun at 23:05
  Run = Differential 2nd-5th sun at 23:05
  Run = Incremental mon-sat at 23:05
}

17 Troubleshooting Common Issues

Check Bacula service logs for errors:

sudo tail -f /var/log/bacula/bacula.log
sudo tail -f /var/log/bacula/bacula-dir.log
sudo tail -f /var/log/bacula/bacula-sd.log
sudo tail -f /var/log/bacula/bacula-fd.log

Common issues and solutions:

Issue Solution
Director cannot connect to Storage Daemon Check passwords match in both configurations
Jobs stuck in waiting状态 Check if Storage Daemon is running and accessible
Permission denied errors Verify Bacula user has access to backup directories
Database connection errors Check MySQL service and database credentials
