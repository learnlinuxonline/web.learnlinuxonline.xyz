RAID Configuration on Ubuntu and RHEL

RAID (Redundant Array of Independent Disks) combines multiple physical disks into a single logical unit to improve performance, reliability, or both. This tutorial covers setting up various RAID levels on both Ubuntu and RHEL systems.

Prerequisites

  • Ubuntu Server 20.04 LTS or newer, or RHEL 8/9
  • Root or sudo privileges
  • Multiple physical disks (at least 2 for RAID 1, 3 for RAID 5, etc.)
  • Backup of any important data (RAID setup will erase disk contents)
  • Basic knowledge of Linux command line and disk management

RAID 0 (Striping)

  • Minimum disks: 2
  • Pros: Improved performance
  • Cons: No redundancy
  • Use case: Temporary data, caching

RAID 1 (Mirroring)

  • Minimum disks: 2
  • Pros: Data redundancy
  • Cons: 50% storage efficiency
  • Use case: Critical data storage

RAID 5 (Striping with Parity)

  • Minimum disks: 3
  • Pros: Good balance of performance and redundancy
  • Cons: Slow write performance
  • Use case: File servers, NAS

RAID 6 (Double Parity)

  • Minimum disks: 4
  • Pros: Can withstand two disk failures
  • Cons: Lower write performance than RAID 5
  • Use case: Large storage arrays

RAID 10 (Mirroring + Striping)

  • Minimum disks: 4
  • Pros: High performance and redundancy
  • Cons: 50% storage efficiency
  • Use case: Databases, application servers

1 Install Required Packages

Update your package list and install mdadm (the RAID management tool):

sudo apt update
sudo apt install mdadm -y

2 Identify Available Disks

List all available disks on your system:

lsblk

Or use fdisk to list disks:

sudo fdisk -l
Note: Make sure to identify the correct disks you want to use for RAID. All data on these disks will be lost during the RAID creation process.

3 Create RAID Array

Create a RAID 1 array (mirroring) with two disks (replace sdb and sdc with your disk identifiers):

sudo mdadm --create --verbose /dev/md0 --level=1 --raid-devices=2 /dev/sdb /dev/sdc

For other RAID levels:

  • RAID 0: --level=0 --raid-devices=2 /dev/sdb /dev/sdc
  • RAID 5: --level=5 --raid-devices=3 /dev/sdb /dev/sdc /dev/sdd
  • RAID 6: --level=6 --raid-devices=4 /dev/sdb /dev/sdc /dev/sdd /dev/sde
  • RAID 10: --level=10 --raid-devices=4 /dev/sdb /dev/sdc /dev/sdd /dev/sde

4 Check RAID Status

Check the status of your RAID array:

cat /proc/mdstat

Get detailed information about the RAID array:

sudo mdadm --detail /dev/md0

5 Create Filesystem on RAID Array

Create a filesystem on the RAID array (ext4 example):

sudo mkfs.ext4 /dev/md0

For XFS filesystem:

sudo mkfs.xfs /dev/md0

6 Mount the RAID Array

Create a mount point and mount the RAID array:

sudo mkdir -p /mnt/raid
sudo mount /dev/md0 /mnt/raid

Make the mount permanent by adding it to /etc/fstab:

echo '/dev/md0 /mnt/raid ext4 defaults,nofail,discard 0 0' | sudo tee -a /etc/fstab

7 Save RAID Configuration

Save the RAID configuration to ensure it's reassembled automatically at boot:

sudo mdadm --detail --scan | sudo tee -a /etc/mdadm/mdadm.conf

Update the initramfs to include RAID support:

sudo update-initramfs -u

8 Monitor RAID Health

Check RAID status regularly:

cat /proc/mdstat

Set up email alerts for RAID events by editing /etc/mdadm/mdadm.conf:

sudo nano /etc/mdadm/mdadm.conf

Add or modify the MAILADDR line:

MAILADDR your-email@example.com

9 RAID Management Commands

Common RAID management commands for both Ubuntu and RHEL:

Command Description
sudo mdadm --detail /dev/md0 Show detailed information about RAID array
sudo mdadm --stop /dev/md0 Stop a RAID array
sudo mdadm --assemble /dev/md0 Assemble (start) a RAID array
sudo mdadm --manage /dev/md0 --add /dev/sdd Add a disk to a RAID array
sudo mdadm --manage /dev/md0 --remove /dev/sdd Remove a disk from a RAID array
sudo mdadm --manage /dev/md0 --fail /dev/sdd Mark a disk as failed
sudo mdadm --grow /dev/md0 --raid-devices=4 Grow a RAID array by adding devices

10 Troubleshooting Common Issues

RAID not assembling at boot:

sudo mdadm --assemble --scan

Recovering a failed disk:

# Mark disk as failed
sudo mdadm --manage /dev/md0 --fail /dev/sdd

# Remove failed disk
sudo mdadm --manage /dev/md0 --remove /dev/sdd

# Add new disk
sudo mdadm --manage /dev/md0 --add /dev/sde

Checking RAID array consistency:

echo check > /sys/block/md0/md/sync_action
cat /proc/mdstat
